Решение задачи комбинированного выведения КА на ГСО
Мотивация
Мы выбрали двухэтапный подход, потому что он естественно следует из самой постановки задачи и позволяет разделить решение на две части разной сложности.

На первом этапе требовалось определить оптимальную промежуточную орбиту после работы разгонного блока. Здесь ключевая особенность задачи в том, что химический участок и малотяговый участок влияют друг на друга противоположным образом. Если разгонный блок выводит аппарат на более “удобную” орбиту, то электроракетному двигателю требуется меньше времени на довыведение, но масса аппарата после отделения уменьшается. Если же промежуточная орбита выбирается более дешёвой для разгонного блока, то стартовая масса КА для ЭРДУ больше, но длительность малотягового перехода возрастает. Поэтому на первом этапе разумно искать компромисс между затратами разгонного блока и временем электрического перелёта.

Для первого этапа был выбран инженерный метод на основе перебора параметров промежуточной орбиты и интерполяции таблицы минимально-временных характеристических скоростей из статьи Konstantinov–Petukhov. Такой подход оказался удачным по нескольким причинам:

он хорошо согласуется с постановкой задачи, где таблица характеристических скоростей дана явно;
он прост в реализации и устойчив численно;
он позволяет сразу строить требуемые графики “масса–время” и зависимости параметров оптимальной промежуточной орбиты от длительности перелёта;
он не требует на первом этапе решать сложную краевую задачу оптимального управления.
На втором этапе мы не стали сразу решать полную задачу оптимального управления через принцип максимума Понтрягина в строгой форме, так как это приводит к существенно более сложной двухточечной краевой задаче с сопряжёнными переменными и высокой чувствительностью к начальным приближениям. Вместо этого был выбран параметризованный закон управления вектором тяги ЭРДУ в локальном орбитальном базисе с последующим численным подбором параметров. Такой подход позволил быстро получить реальную траекторию, удовлетворяющую конечным условиям, а затем улучшить результат за счёт настройки параметров закона управления.

Таким образом, выбранный метод сочетает инженерную простоту, физическую интерпретируемость и практическую работоспособность.

Математическая и физическая модель
Первый этап
На первом этапе движение рассматривалось в модели центрального ньютоновского гравитационного поля Земли. Разгонный блок моделировался в импульсной постановке, а электрический участок — через характеристическую скорость, получаемую интерполяцией табличных данных.

Основные используемые соотношения:

круговая скорость на опорной орбите:

[ v_0 = \sqrt{\frac{\mu}{r_0}} ]

большая полуось промежуточной орбиты:

[ a = \frac{r_p + r_a}{2} ]

скорость в перицентре промежуточной орбиты:

[ v_p = \sqrt{\mu\left(\frac{2}{r_p} - \frac{1}{a}\right)} ]

разгонный блок выполняет один совмещённый импульс, одновременно меняющий модуль скорости и наклонение:

[ \Delta V_{RB} = \sqrt{v_0^2 + v_p^2 - 2v_0v_p\cos(\Delta i)} ]

масса после работы разгонного блока вычислялась по уравнению Циолковского:

[ m_1 = m_0 \exp\left(-\frac{\Delta V}{c}\right) ]

Для участка ЭРДУ использовалась безразмерная таблица минимально-временных характеристических скоростей перехода с эллиптической орбиты на круговую. Радиусы перицентра и апоцентра нормировались на радиус ГСО, а затем по таблице выполнялась трёхмерная линейная интерполяция.

После этого характеристическая скорость ЭРДУ переводилась в размерный вид:

[ \Delta V_{EP} = V^*{ch} \cdot v{circ,GEO} ]

Конечная масса КА на ГСО снова находилась по уравнению Циолковского, а время перелёта — по постоянному расходу массы:

[ \dot m = -\frac{T}{c} ]

[ t = \frac{m_{start} - m_{final}}{|\dot m|} ]

Второй этап
На втором этапе движение КА моделировалось напрямую в инерциальной геоцентрической системе координат в декартовых переменных.

Использовалась система уравнений:

[ \dot{\mathbf r} = \mathbf v ]

[ \dot{\mathbf v} = -\frac{\mu}{r^3}\mathbf r + \frac{T}{m}\mathbf u ]

[ \dot m = -\frac{T}{c} ]

где:

(\mathbf r) — радиус-вектор КА,
(\mathbf v) — вектор скорости,
(m) — масса КА,
(\mathbf u) — единичный вектор направления тяги,
(T) — тяга ЭРДУ,
(c = I_{sp}g_0).
Для анализа состояния траектории на каждом шаге вычислялись оскулирующие элементы орбиты:

большая полуось,
эксцентриситет,
наклонение.
Они определялись через:

удельную орбитальную энергию,
вектор углового момента,
вектор эксцентриситета.
Например:

[ \varepsilon = \frac{v^2}{2} - \frac{\mu}{r} ]

[ a = -\frac{\mu}{2\varepsilon} ]

[ \mathbf e = \frac{\mathbf v \times \mathbf h}{\mu} - \frac{\mathbf r}{r} ]

[ e = |\mathbf e| ]

[ i = \arccos\left(\frac{h_z}{|\mathbf h|}\right) ]

Для закона управления использовался локальный орбитальный базис:

радиальное направление,
касательное направление,
нормаль к плоскости орбиты.
Ключевая идея второго этапа состояла в том, чтобы разделить управление на две части:

внеплоскостная компонента уменьшает наклонение;
внутриплоскостная компонента изменяет большую полуось и эксцентриситет.
Для кругования орбиты использовалась фазовая логика по истинной аномалии. Это позволило естественно различать движение вблизи перицентра и апоцентра и эффективнее уменьшать эксцентриситет.

Hackathon.Stage1

Техническая реализация первого этапа
1. Таблица характеристических скоростей
В проект была вручную перенесена Table 1 из статьи Konstantinov–Petukhov. Таблица хранится как трёхмерный массив по аргументам:

радиус апоцентра,
радиус перицентра,
наклонение.
Для получения значения между узлами реализована трёхмерная линейная интерполяция.

2. Оценка одной промежуточной орбиты
Для каждой тройки параметров ((r_p, r_a, i)) выполнялись следующие шаги:

вычисление скорости на опорной орбите и в перицентре промежуточной орбиты;
вычисление (\Delta V) разгонного блока как одного совмещённого импульса;
вычисление массы после работы разгонного блока;
вычитание массы отделяемого разгонного блока;
нормировка параметров промежуточной орбиты и интерполяция табличной характеристики для ЭРДУ;
вычисление конечной массы на ГСО;
вычисление длительности перелёта.
3. Поиск решения
Был реализован перебор по сетке параметров промежуточной орбиты:

(r_p),
(r_a),
(i).
Все допустимые решения сохранялись, после чего по временным интервалам строилась оптимальная огибающая: для каждого интервала выбиралось решение с максимальной массой на ГСО.

Итоговое решение первого этапа выбиралось как решение с минимальной длительностью перелёта при условии доставки на ГСО не менее требуемой массы.

Техническая реализация второго этапа
1. Представление состояния
Состояние КА задаётся классом, содержащим:

время,
радиус-вектор,
вектор скорости,
массу.
Для векторной алгебры реализован собственный тип Vector3d, который содержит:

норму,
нормировку,
скалярное произведение,
векторное произведение,
арифметические операции.
2. Начальные условия
Начальные условия второго этапа строились из найденной на первом этапе промежуточной орбиты. Для этого использовался переход от кеплеровых элементов к декартовым координатам:

вычисление (a, e, p),
построение вектора положения и скорости в системе PQW,
поворот в инерциальную систему через (\Omega, i, \omega).
3. Численное интегрирование
Для интегрирования системы ОДУ был реализован метод Рунге–Кутты четвёртого порядка (RK4). Этот метод оказался достаточно точным и устойчивым для задачи с постоянной малой тягой.

Для основного поиска использовался более крупный шаг интегрирования, а для финального прогона лучшего набора параметров — уменьшенный шаг.

4. Закон управления
Вектор тяги строился в локальном орбитальном базисе:

(\hat r),
(\hat t),
(\hat n).
Закон управления параметризовался величинами:

Ke,
Ka,
ISwitchDeg,
INormalMax.
Управление имело двухрежимную структуру:

пока наклонение превышает порог ISwitchDeg, часть тяги направляется вне плоскости орбиты для уменьшения наклонения;
после достижения порога внеплоскостная составляющая отключается, и вся тяга используется для доводки большой полуоси и эксцентриситета.
Внутриплоскостная компонента зависела от истинной аномалии, что позволило учитывать фазу движения по орбите и эффективно круговать орбиту.

5. Подбор параметров закона управления
Поиск лучшего управления выполнялся в два шага:

грубый перебор по редкой сетке параметров;
локальное уточнение в окрестности лучших найденных значений.
Для каждого набора параметров выполнялся полный численный прогон. Для каждого прогона сохранялись:

достигнута ли целевая орбита;
время перелёта;
масса на ГСО;
конечные (a, e, i).
Лучшее решение выбиралось среди допустимых по минимальному времени перелёта.

Итог
Предложенный подход позволил:

устойчиво решить первый этап задачи через перебор промежуточных орбит и табличную оценку малотягового перехода;
построить реализуемую траекторию второго этапа с фазово-зависимым законом управления;
получить конечные графики, CSV-файлы и численные характеристики, пригодные для отчёта и презентации.
Основное достоинство решения — в том, что оно сочетает:

физическую интерпретируемость,
понятную инженерную логику,
численную устойчивость,
удобство последующей доработки и анализа.
